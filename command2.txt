GPU実行コマンド集

目的:
- `CA-FM last-mile` の `direct` と `two_stage` を GPU で再現する
- 2軌道での比較、`two_stage` の小さな sweep、solver trace 収集まで回せるようにする
- コマンドだけでなく、何を比較しているかもこのファイルだけで分かるようにする

前提:
- `cuda124` を使う
- conda 環境は `asrti`
- Newton / Warp は `--device cuda:0`
- GPU 比較実験は並列ではなく serial 実行を推奨
- 実行本体は `AS-RTI-acados/test_adjflow_cloth_lastmile.py`

推奨既定値:
- `--lastmile-mode two_stage`
- `--corner-source auto`
- `--corner-min-confidence 0.80`
- `--lastmile-ramp-sec 0.50`
- `solver-flow` を 1 値に統一するなら `--solver-flow-mix 0.20`
- `140 frame` の短尺 smoke だけなら `--solver-flow-mix 0.50` が速い条件もある
- `num_frames` で自動切替したいなら `--solver-flow-mix-policy auto --solver-flow-mix-short 0.50 --solver-flow-mix-long 0.20 --solver-flow-mix-short-max-frames 140`
- short の `residual fallback` 既定値は `--solver-flow-residual-threshold 1.0 --solver-flow-residual-min-scale 0.20 --solver-flow-residual-mode max_all`
- `threshold=1.0` は `angle47_dist70 / 140 frames` の rerun でも baseline 比 `prep mean -1.34ms`, `prep p99 -9.04ms`, `final corner delta 0.176mm`
- `--solver-flow-residual-mode stat_only` はこの系では不要。`res_stat` が支配的なので `max_all` と実質同じになる
- ただし現時点の `auto` は convenience 用。長尺 benchmark は固定 `--solver-flow-mix 0.20` の方が読みやすい
- `residual fallback` は short では有効だが、long ではまだ fixed `0.20` の完全代替ではない
- `learned corrector` は exploratory。short smoke 1本では `corrector_mix=0.20` で baseline 比 `prep mean -1.64ms`, `prep p99 -11.72ms`, `final corner delta 0.056mm`
- 詳細比較は `adjflow/kiroku/20260307_solver_flow_cross_horizon_mix020.md` と `adjflow/kiroku/20260307_solver_flow_auto_mix_policy.md` と `adjflow/kiroku/20260307_solver_flow_residual_fallback.md` と `adjflow/kiroku/20260307_solver_flow_residual_sweep_and_modes.md`
- まずは `trajectory_angle45_dist50.npy` で確認し、その後 `angle47_dist70.npy` に広げる

重要フラグ:
- `--corner-source vision`
  深度ベース detector で corner を使う
- `--corner-source oracle`
  cloth fold corner の GT を使う
- `--corner-source auto`
  confidence が閾値未満なら oracle fallback
- `--lastmile-mode direct`
  `grasp_pos` をそのまま Phase 1-4 の target に差し替える旧方式
- `--lastmile-mode two_stage`
  `trust / ramp / z_track` を持つ plan を作り、solver 側で時間方向に target を blend する方式
- `--lastmile-ramp-sec`
  `two_stage` の solver-side blending の基準時間
- `--corner-min-confidence`
  `auto` で vision corner を採用する最低 confidence
- `--save-solver-trace`
  horizon の `x / u / yref / target` などを trace として保存する
