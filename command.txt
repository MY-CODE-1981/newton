# 布広げデータ保存
  Created: AS-RTI-acados/fold_cloth_newton.py (~290 lines)                                                           
                                                                                                                     
  What it does                                                                                                       
                                                                                                                     
  Simulates cloth corner folding using Newton physics with kinematic vertex control (no robot gripper). Generates NPY
   trajectory files compatible with the existing pipeline (_apply_fold_from_npy).

  Architecture

  - Scene: Cloth mesh from OBJ + ground plane, VBD solver only (no robot/Featherstone)
  - Grasp mechanism: Pins corner vertices (inv_mass=0) and updates their positions each frame along the trajectory
  - 3-phase trajectory: Lift (smoothstep 0-15%) → Travel (linear XY, 15-85%) → Descend (smoothstep 85-100%)
  - Follows Newton Example pattern: __init__/step()/render() with newton.examples.run()

  Key parameters

  ┌──────────────────┬───────────────────────────┬──────────────────────────────────────────────────┐
  │       Flag       │          Default          │                   Description                    │
  ├──────────────────┼───────────────────────────┼──────────────────────────────────────────────────┤
  │ --angle          │ 45                        │ Fold direction in degrees (0=+X, 90=+Y)          │
  ├──────────────────┼───────────────────────────┼──────────────────────────────────────────────────┤
  │ --distance       │ 70                        │ Travel distance as % of cloth diagonal (~0.332m) │
  ├──────────────────┼───────────────────────────┼──────────────────────────────────────────────────┤
  │ --corner         │ 0                         │ Corner vertex (0/48/2254/2302)                   │
  ├──────────────────┼───────────────────────────┼──────────────────────────────────────────────────┤
  │ --pin-edge       │ off                       │ Pin yellow-green edge (47 vertices)              │
  ├──────────────────┼───────────────────────────┼──────────────────────────────────────────────────┤
  │ --num-frames     │ 600                       │ Simulation frames                                │
  ├──────────────────┼───────────────────────────┼──────────────────────────────────────────────────┤
  │ --save-interval  │ 4                         │ Record every N frames                            │
  ├──────────────────┼───────────────────────────┼──────────────────────────────────────────────────┤
  │ --trajectory-dir │ ./cloth_trajectory_newton │ Output directory                                 │
  └──────────────────┴───────────────────────────┴──────────────────────────────────────────────────┘

  Output

  - fold/trajectory_angle{A}_dist{D}.npy — shape (N_saved, 2303, 3), local coords
  - unfold/trajectory_angle{A}_dist{D}.npy — time-reversed version
  - trajectory_angle{A}_dist{D}_metadata.json — parameters + grasp indices

GPU実行コマンド集

目的:
- CA-FM last-mile 実験を GPU で再現するための実行メモ
- `vision` と `oracle` の比較、`auto` fallback の確認をすぐ回せるようにする
- どのコマンドが何の比較なのかを、このファイルだけで分かるようにする

前提:
- `cuda124` を使う
- conda環境は `asrti`
- Newton / Warp は `--device cuda:0`
- 比較実験は並列ではなく serial 実行を推奨
- 現在の `simple_corner_detector.py` は tuned radial version
- 実行本体は `AS-RTI-acados/test_adjflow_cloth_lastmile.py`

重要フラグの意味:
- `--corner-source vision`
  画像ベース corner detector だけを使う。oracle fallback しない。
- `--corner-source oracle`
  シミュレーションの live cloth corner を直接使う。基準線。
- `--corner-source auto`
  vision を試し、confidence が閾値未満なら oracle fallback する。
- `--corner-min-confidence X`
  auto / vision で受理する最小 confidence。
- `--cafm-inject-time 2.0`
  `t=2.0s` で corner を観測し、CA-FM の grasp を MPC target に注入する。
- `--num-frames 240`
  注入後に少し先まで挙動を見たい本番寄りの長さ。
- `--num-frames 140`
  注入の成否と grasp 近辺だけを見る短めの確認。
- `--output-dir /tmp/...`
  出力先。主に以下が保存される。
  - `log_lastmile_cafm.npz`
  - `log_lastmile_cafm_cameras.npz`
  - `metrics_lastmile_cafm.npz`
